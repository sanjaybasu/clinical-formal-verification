Inputs:

  stiff_neck: boolean
  focal_weakness: boolean
  fever: boolean
  pallor: boolean
  rash: boolean
  dizziness: boolean
  slurred_speech: boolean
Outputs:
  acuity: one of [self_care, routine, urgent, emergent] (ordered low to high)
Conflict resolution: priority (per output, the matching rule with the highest priority wins; ties broken by order)
Rules:
  [r1, priority 3] IF palpitations THEN acuity := emergent
  [r2, priority 3] IF focal_weakness THEN acuity := emergent
  [r3, priority 2] IF rash THEN acuity := urgent

  [r8, priority 2] IF severe_headache THEN acuity := urgent
  [r9, priority 2] IF stiff_neck THEN acuity := urgent
Default (for any output left unset): acuity := routine

Property (monotonicity): for any input, if dizziness is set to true while every other input is unchanged, then acuity must be no lower than before (enum order is the listed order).
Plain-language intent: setting dizziness true never lowers acuity

Reason carefully about interactions between rules, priorities, and defaults. Then answer whether the property holds or is violated.
