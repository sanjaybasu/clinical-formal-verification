  low_ejection_fraction: boolean
  g6pd_deficiency: boolean
  peptic_ulcer: boolean
  bradycardia: boolean
  asthma: boolean
  pregnancy: boolean
  renal_impairment: boolean
Outputs:
  rec_drug_a: boolean
  rec_drug_b: boolean
  rec_drug_c: boolean
Conflict resolution: priority (per output, the matching rule with the highest priority wins; ties broken by order)
Rules:
  [r1, priority 40] IF asthma THEN rec_drug_c := true
  [r2, priority 40] IF pregnancy THEN rec_drug_c := true
  [r3, priority 40] IF renal_impairment THEN rec_drug_c := true
  [r4, priority 50] IF (qt_prolongation and hypertension and low_ejection_fraction and g6pd_deficiency and peptic_ulcer and bradycardia) THEN rec_drug_a := true

Default (for any output left unset): rec_drug_a := false, rec_drug_b := false, rec_drug_c := false

Property (mutual exclusion): no input may make any of these jointly true: (rec_drug_a and rec_drug_b).
Plain-language intent: no input recommends both drug a and drug b

Reason carefully about interactions between rules, priorities, and defaults. Then answer whether the property holds or is violated.
