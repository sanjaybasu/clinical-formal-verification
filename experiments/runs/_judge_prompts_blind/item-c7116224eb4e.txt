  pallor: boolean
  dizziness: boolean
  chest_pain: boolean
  fever: boolean
Outputs:
  acuity: one of [self_care, routine, urgent, emergent] (ordered low to high)
Conflict resolution: priority (per output, the matching rule with the highest priority wins; ties broken by order)

  [r2, priority 3] IF back_pain THEN acuity := emergent
  [r3, priority 3] IF severe_headache THEN acuity := emergent
  [r4, priority 3] IF pallor THEN acuity := emergent

  [r6, priority 2] IF dizziness THEN acuity := urgent
  [r7, priority 2] IF vomiting THEN acuity := urgent

Property (monotonicity): for any input, if chest_pain is set to true while every other input is unchanged, then acuity must be no lower than before (enum order is the listed order).
Plain-language intent: setting chest_pain true never lowers acuity

Reason carefully about interactions between rules, priorities, and defaults. Then answer whether the property holds or is violated.
